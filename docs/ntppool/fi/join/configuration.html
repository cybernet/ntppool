Asetussuosituksia pooliin liittyville palvelimille

support.ntp.org:in tukioisiossa on paljon hyödyllistä tietoa.

Jos haluat vain käyttää poolia, katso kuinka käytän poolia.

comp.protocols.time.ntp-uutisryhmä on paras paikka etsiä apua ntpd-ohjelmaan liittyen.

Alla on joitain huomioon otettavia seikkoja jos aiot liittää palvelimesi pooliin.

Management queries

Make the default configuration be to not allow "management queries". For ntpd this will be adding the "noquery" option to the default "restrict" lines, for example:

[% INCLUDE "join/configuration/restrict-default.html" %]

To allow commands like "ntpq -c pe" to work from localhost you can add:

[% INCLUDE "join/configuration/restrict-localhost.html" %]

Käytä noin viittä palvelinta

Toimiakseen kunnolla ntpd tarvitsee vähintään kolme palvelinta joihin synkronoitua ("Mies jolla on kello tietää paljonko se on. Mies jolla on kaksi kelloa ei voi ikinä olla varma.")

Suosittelemme pooliin kuuluville palvelimille vähintään neljän mutta ei enempää kuin seitsemän palvelimen käyttämistä.

Älä käytä *.pool.ntp.org:in palvelimia

Ironista on että varmistaaksesi parhaan mahdollisen palvelun tason, liitäessäsi palvelimesi pooliin sinun ei tule käyttää *.pool.ntp.org-aliaksia asetustiedostossasi.

Poolin luotettavuuden varmistamiseksi poolin operaattoreiden tulisi valita käsin hyvät paikalliset (verkkomielessä) palvelimet. NTP.org:in wikissä on lista julkisista palvelimista.

Käytä standardia ntpd:tä

Suosimme ohjelmiston monimuotoisuutta, mutta merkittävä osuus "se ei toimi" -kysymyksistä joita meille esitetään liittyy muihin kuin ntpd:hen.

Voit käyttää poolia millä tahansa NTP:tä ymmärtävällä ohjelmistolla, mutta mikäli aiot liittyä pooliin, suosittelemme käyttämään ntpd:tä.

Älä käytä LOCAL-kelloajuria

Poolin palvelimilla ei pitäisi olla LOCAL-kelloajuria konfiguroituna.
